# Specter Database Schema

This document outlines the detailed relational database tables required for the Specter platform. It is designed for PostgreSQL (ideal for Supabase) and structured to support the core operational flow, from user accounts down to individual persona click events.

---

## 1. Core Platform Tables

### `users`
Stores account and billing information for human users (founders/teams).
*   `id` (UUID, Primary Key)
*   `email` (String, Unique)
*   `name` (String, Nullable)
*   `plan_tier` (Enum: `free`, `pro`, `team` ‚Äî Default: `free`)
*   `stripe_customer_id` (String, Nullable)
*   `created_at` (Timestamp)
*   `updated_at` (Timestamp)

### `projects`
Represents a single web application being tested by a User.
*   `id` (UUID, Primary Key)
*   `user_id` (UUID, Foreign Key -> `users.id`)
*   `name` (String) ‚Äî e.g., "Specter Marketing Site"
*   `target_url` (String) ‚Äî The base URL to begin testing
*   `requires_auth` (Boolean, Default: false)
*   `auth_credentials` (Text, Nullable) ‚Äî Encrypted JSON containing test usernames/passwords or session cookies.
*   `created_at` (Timestamp)
*   `updated_at` (Timestamp)

---

## 2. Testing Configuration Tables

### `persona_configs`
The predefined attribute configurations that dictate how a synthetic user behaves.
*   `id` (UUID, Primary Key)
*   `project_id` (UUID, Foreign Key -> `projects.id`)
*   `name` (String) ‚Äî e.g., "Low-Tech Skeptic"
*   `age_range` (String) ‚Äî e.g., "50-65"
*   `geolocation` (String) ‚Äî e.g., "US-Midwest"
*   `tech_literacy` (Enum: `low`, `medium`, `high`)
*   `goal_prompt` (Text) ‚Äî Explicit instruction: "Try to find the pricing page and sign up for a pro account."
*   `ai_system_prompt` (Text) ‚Äî The deeply expanded psychological profile generated by the framework before testing begins.
*   `created_at` (Timestamp)
*   `updated_at` (Timestamp)

### `test_runs`
A single deployment batch. Represents the event of a User clicking "Run Test Now".
*   `id` (UUID, Primary Key)
*   `project_id` (UUID, Foreign Key -> `projects.id`)
*   `status` (Enum: `pending`, `running`, `completed`, `failed`)
*   `started_at` (Timestamp, Nullable)
*   `completed_at` (Timestamp, Nullable)
*   `created_at` (Timestamp)

---

## 3. Session & Execution Tables

### `persona_sessions`
An active browser instance for a specific persona tied to a specific test run.
*   `id` (UUID, Primary Key)
*   `test_run_id` (UUID, Foreign Key -> `test_runs.id`)
*   `persona_config_id` (UUID, Foreign Key -> `persona_configs.id`)
*   `status` (Enum: `queued`, `running`, `completed`, `abandoned`, `error`)
*   `exit_reason` (Text, Nullable) ‚Äî e.g., "goal_achieved", "rage_quit", "timeout", "captcha_blocked"
*   `started_at` (Timestamp)
*   `completed_at` (Timestamp, Nullable)
*   `created_at` (Timestamp)

### `session_logs`
The granular, step-by-step history of what the persona did, saw, and felt. This is the heaviest table and the core data engine for reports.
*   `id` (UUID, Primary Key)
*   `session_id` (UUID, Foreign Key -> `persona_sessions.id`)
*   `step_number` (Integer) ‚Äî Sequential ordering (1, 2, 3...)
*   `current_url` (String) ‚Äî The exact URL the browser was on during this step
*   `screenshot_url` (String) ‚Äî Cloud Object Storage (S3/R2) URL to the image captured
*   `emotion_tag` (Enum: `neutral`, `confusion`, `frustration`, `delight`)
*   `emotion_score` (Integer 1-10) ‚Äî The intensity of the emotion
*   `inner_monologue` (Text) ‚Äî The LLM's raw thoughts ("I don't see the checkout button anywhere...")
*   `action_taken` (JSONB) ‚Äî Structured data of the executed Playwright command:
    *   *Example:* `{"type": "click", "target_id": "submit-btn", "x": 450, "y": 920}`
    *   *Example:* `{"type": "type", "target_id": "email-input", "value": "test@demo.com"}`
    *   *Example:* `{"type": "scroll", "direction": "down", "pixels": 500}`
*   `created_at` (Timestamp)

---

## 4. Aggregation Tables

### `reports`
The final synthesized output generated once a `TestRun` concludes.
*   `id` (UUID, Primary Key)
*   `test_run_id` (UUID, Foreign Key -> `test_runs.id`, Unique)
*   `product_opportunity_score` (Integer 0-100) ‚Äî High-level metric of success
*   `executive_summary` (Text) ‚Äî LLM-synthesized overall summary of the cohort's experience
*   `funnel_completion_rate` (Decimal) ‚Äî Percentage of personas that achieved their configured goal
*   `heatmap_data_url` (String) ‚Äî S3/R2 link to a JSON blob of aggregated X/Y coordinates to render UI heatmaps (keeps the DB lean).
*   `created_at` (Timestamp)
