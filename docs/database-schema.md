# Database Schema

Supabase (PostgreSQL). All tables have RLS enabled. Auth uses Clerk JWTs — `auth.jwt() ->> 'sub'` resolves to the Clerk user ID.

---

## Enums

| Enum | Values |
|---|---|
| `plan_tier` | `free`, `pro`, `team` |
| `tech_literacy` | `low`, `medium`, `high` |
| `test_run_status` | `pending`, `running`, `completed`, `failed` |
| `session_status` | `queued`, `running`, `completed`, `abandoned`, `error` |
| `emotion_tag` | `neutral`, `confusion`, `frustration`, `delight`, `satisfaction`, `curiosity`, `surprise`, `boredom`, `disappointment` |

### `projects`

| Column | Type | Notes |
|---|---|---|
| `id` | `UUID` PK | |

### `persona_sessions`

One session = one persona running through the site. A test run has N sessions (one per persona).

| Column | Type | Notes |
|---|---|---|
| `id` | `UUID` PK | |
| `test_run_id` | `UUID` FK → `test_runs.id` | |
| `persona_config_id` | `UUID` FK → `persona_configs.id` | |
| `status` | `session_status` | Default `queued` |
| `exit_reason` | `TEXT` | Why session ended |
| `execution_mode` | `TEXT` | `autonomous` \| `manual` |
| `is_paused` | `BOOLEAN` | Manual mode — session waiting for next step |
| `step_requested` | `BOOLEAN` | Manual mode — user clicked "next step" |
| `live_status` | `TEXT` | Real-time status string shown in UI |
| `started_at` | `TIMESTAMPTZ` | |
| `completed_at` | `TIMESTAMPTZ` | |
| `created_at` | `TIMESTAMPTZ` | |

**Realtime:** Enabled via `supabase_realtime` publication — UI subscribes to live updates.
**Index:** `idx_persona_sessions_paused` on `is_paused WHERE is_paused = true`
**RLS:** User can view/insert sessions that belong to their test runs.

### `session_logs`

One row per action step taken by the persona during a session.

| Column | Type | Notes |
|---|---|---|
| `id` | `UUID` PK | |
| `session_id` | `UUID` FK → `persona_sessions.id` | |
| `step_number` | `INTEGER` | Ordered step index within the session |
| `current_url` | `TEXT` | URL at time of step |
| `screenshot_url` | `TEXT` | S3 URL of screenshot at this step |
| `emotion_tag` | `emotion_tag` | Emotion detected at this step |
| `emotion_score` | `INTEGER` | 1–10 intensity score |
| `inner_monologue` | `TEXT` | Persona's reasoning / UX feedback |
| `action_taken` | `JSONB` | Full action object — see below |
| `created_at` | `TIMESTAMPTZ` | |

**`action_taken` JSONB shape:**
```json
{
  "type": "click | type | scroll | wait | complete | fail | skip_node | system",
  "selector": "xpath=...",
  "text": "Subscribe",
  "reasoning": "first-person monologue",
  "emotional_state": "curiosity",
  "emotional_intensity": 0.7,
  "specific_emotion": "intrigued",
  "ux_feedback": "CTA is unclear",
  "proposed_solution": "Change button label to...",
  "possible_paths": ["...", "..."],
  "info": "session_started | browser_init | scan_top | session_completed | ...",
  "local_screenshot_path": "https://s3.../screenshots/...",
  "click_coords": { "x": 120, "y": 340, "w": 80, "h": 32 }
}
```

**Realtime:** Enabled — UI streams new steps live during a session.
**RLS:** User can view logs that belong to their sessions.

### `reports`

One report per test run. Generated after all sessions complete.

| Column | Type | Notes |
|---|---|---|
| `id` | `UUID` PK | |
| `test_run_id` | `UUID` UNIQUE FK → `test_runs.id` | |
| `product_opportunity_score` | `INTEGER` | 0–100 overall UX score |
| `executive_summary` | `TEXT` | LLM-generated summary |
| `funnel_completion_rate` | `DECIMAL(5,2)` | % of sessions that reached goal |
| `heatmap_data_url` | `TEXT` | |
| `report_data` | `JSONB` | Pre-calculated aggregations (emotion stats, session scores, etc.) |
| `created_at` | `TIMESTAMPTZ` | |

**RLS:** User can view reports for their test runs.

---

### `ai_caches`

Cache for LLM calls that are expensive to repeat (archetype/persona suggestions).

| Column | Type | Notes |
|---|---|---|
| `id` | `UUID` PK | |
| `cache_key` | `TEXT` UNIQUE | Hash of the input (URL + prompt) |
| `payload` | `JSONB` | Cached LLM response |
| `cache_type` | `TEXT` | e.g. `archetypes`, `personas` |
| `created_at` | `TIMESTAMPTZ` | |

**RLS:** Any authenticated user can read/insert (global cache, not user-scoped).

---

## Entity Relationship

```
users
 └── projects
      └── test_runs
           ├── persona_sessions
           │    └── session_logs
           └── reports

persona_configs ──── persona_sessions
                     (can exist without a project — library personas)

ai_caches (standalone, global cache)
```
